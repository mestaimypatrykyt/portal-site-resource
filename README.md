# portal-site-resource

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面可作为门户站点或资源页面的基础素材，支持直接引用或嵌入到其他项目中。仓库内容不针对特定网站或域名，旨在提供通用的页面资源存储与分发能力。

## 目录结构

```
portal-site-resource/
├── pages/          # 存放独立的 HTML 页面文件
├── assets/         # 页面引用的静态资源（CSS、JS、图片等）
├── templates/      # 可复用的页面模板或示例
└── README.md       # 本文件
```

- **pages/**：每个子目录或文件对应一个完整的 HTML 页面，可直接通过浏览器访问。
- **assets/**：包含页面所需的公共或私有资源，按类型或用途组织。
- **templates/**：提供页面结构、组件或样式示例，便于快速生成新页面。

## 页面归档说明

- 所有页面均为独立 HTML 文件，无外部依赖（除非在 `assets/` 中显式提供）。
- 页面内容不包含特定品牌、服务或推广信息，仅作为通用资源使用。
- 归档时建议在文件名或目录名中标注页面功能或版本，如 `landing-v1.html`。

## 维护说明

- 新增页面请放入 `pages/` 目录，并确保引用的资源路径正确。
- 若页面涉及外部资源（如字体库、图标库），建议下载后放入 `assets/` 目录，避免依赖外部链接。
- 修改或更新页面时，请保留旧版本（可移至 `pages/archive/` 目录）以便回溯。
- 欢迎提交 Pull Request 或 Issue 以改进页面内容或结构。

## 使用方式

1. 克隆或下载本仓库。
2. 直接打开 `pages/` 下的 HTML 文件即可预览。
3. 也可将 `pages/` 或 `assets/` 部署到任意静态文件服务器，通过 URL 访问对应页面。
